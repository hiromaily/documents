# サービス比較

## 小規模なサービス向け、インフラ構築に伴う作業コストの低さ（運用の簡便さ）

- AWS Lambda
- AWS App Runner
- SWS Lightsail
- AWS Elastic Beanstalk
- AWS Amplify
- AWS Fargate

| サービス名            | 運用コスト | インフラ管理 | スケーリング | カスタマイズ | 簡便さ     | 適したユースケース                                         | 各サービスの説明                                                                               | 利用可能な言語の制約                                             |
| --------------------- | ---------- | ------------ | ------------ | ------------ | ---------- | ---------------------------------------------------------- | ---------------------------------------------------------------------------------------------- | ---------------------------------------------------------------- |
| AWS Lambda            | 非常に低い | 非常に低い   | 自動         | 低い         | 非常に高い | イベント駆動の小規模から中規模のアプリ、マイクロサービス   | コードをアップロードするだけでサーバレスで実行可能                                             | 制約あり（サポートされているランタイム）                         |
| AWS App Runner        | 低い       | 低い         | 自動         | 中程度       | 高い       | 簡単デプロイとスケーリングが求められる Web アプリや API    | ソースコードやコンテナを簡単にデプロイ、スケーリング                                           | 制約なし（ソースコードや Docker イメージに依存）                 |

## 説明

**AWS Lambda**

- **運用コスト**: 非常に低い
- **インフラ管理**: 非常に低い（完全サーバーレス）
- **スケーリング**: 自動
- **カスタマイズ**: 低い（イベント駆動で設定が簡単だが制約あり）
- **簡便さ**: 非常に高い
- **適したユースケース**: イベント駆動アプリ、小規模から中規模のマイクロサービス
- **各サービスの説明**: コードをアップロードするだけでサーバレスで実行可能
- **利用可能な言語の制約**: 制約あり（サポートされているランタイム）

**AWS App Runner**

- **運用コスト**: 低い
- **インフラ管理**: 低い（サーバーレスに近い）
- **スケーリング**: 自動
- **カスタマイズ**: 中程度（ある程度のカスタマイズが可能）
- **簡便さ**: 高い
- **適したユースケース**: 簡単デプロイとスケーリングが求められる Web アプリや API
- **各サービスの説明**: ソースコードやコンテナを簡単にデプロイ、スケーリング
- **利用可能な言語の制約**: 制約なし（ソースコードや Docker イメージに依存）

**Amazon Lightsail**

- **運用コスト**: 中
- **インフラ管理**: 中（シンプルな初期設定と管理）
- **スケーリング**: 手動（スケールアップ/スケールダウンは手動）
- **カスタマイズ**: 中程度（基本的なカスタマイズが可能）
- **簡便さ**: 中
- **適したユースケース**: 小規模から中規模の Web サイト、ブログ、テスト環境
- **各サービスの説明**: 簡単な UI で仮想サーバー、DB、ストレージを管理
- **利用可能な言語の制約**: 制約なし（仮想サーバー上で必要なソフトウェアをインストール可能）

**AWS Elastic Beanstalk**

- **運用コスト**: やや高い
- **インフラ管理**: 高い（多言語対応とカスタマイズ性が高いが、設定に手間がかかる）
- **スケーリング**: 自動
- **カスタマイズ**: 高い（詳細な設定と制御が可能）
- **簡便さ**: 中
- **適したユースケース**: 中規模から大規模のアプリケーション、多言語対応が必要な場合
- **各サービスの説明**: インフラの多くを自動化してくれる多言語対応プラットフォーム
- **利用可能な言語の制約**: 制約あり（サポートされているプラットフォーム）

**AWS Amplify**

- **運用コスト**: 低い
- **インフラ管理**: 非常に低い（フルマネージド）
- **スケーリング**: 自動
- **カスタマイズ**: 低い
- **簡便さ**: 非常に高い
- **適したユースケース**: Web およびモバイルアプリのフロントエンドとバックエンド
- **各サービスの説明**: スケーラブルなフルマネージドの Web およびモバイルアプリの構築、デプロイ
- **利用可能な言語の制約**: 制約なし（フロントエンド向けの技術スタックに依存）

**AWS Fargate**

- **運用コスト**: 中程度
- **インフラ管理**: 中程度（コンテナオーケストレーションのためのサーバレスオプション）
- **スケーリング**: 自動
- **カスタマイズ**: 高い（詳細な設定と制御が可能）
- **簡便さ**: 中
- **適したユースケース**: コンテナ化されたアプリケーション、マイクロサービス
- **各サービスの説明**: コンテナオーケストレーションのためのサーバレスオプション。Amazon ECS または AWS EKS で利用可能
- **利用可能な言語の制約**: 制約なし（コンテナ内で使用される言語に依存）
